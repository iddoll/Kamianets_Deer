# Kamianets Deer

Мобільний веб-хаб для запуску WebGL-ігор (Unity) під час екскурсій у Кам'янці-Подільському.

## Швидкий старт

```bash
npm install
npm run dev
```

- **ПК:** `http://localhost:5173/`
- **Телефон** (та сама Wi‑Fi): рядок **Network** у терміналі, наприклад `http://192.168.0.175:5173/`

Не пишіть буквально `192.168.x.x` — це лише приклад.

**Важливо:** після зміни на HTTPS сайт на `http://` не відкривався. Зараз знову **`npm run dev` = http**. Для GPS на телефоні — окремо `npm run dev:https`.

### Телефон не відкриває сайт?

1. **Той самий Wi‑Fi** — не мобільний інтернет на телефоні.
2. **Не гостьова мережа** — на роутері часто заборонено зв’язок між пристроями (AP isolation).
3. **Windows Firewall** — найчастіча причина. PowerShell **як адміністратор**:
   ```powershell
   cd D:\GitHub\Kamianets_Deer
   .\scripts\open-firewall-port-5173.ps1
   ```
4. Перезапустіть `npm run dev` і знову відкрийте URL з рядка **Network**.
5. На iPhone: саме **http://** (не https), у Safari в адресному рядку.
6. Вимкніть VPN на телефоні та ПК.
7. Якщо ПК по **кабелю**, а телефон по **Wi‑Fi** — переконайтесь, що це одна підмережа (той самий роутер).

Перевірка: на телефоні має відкритись `http://192.168.0.175:5173/` (ваш IP може відрізнятись).

## Геолокація (без змін Unity)

Кожна гра прив’язана до зони в `src/config/locations.ts`:

| Гра | Локація | Координати (орієнтовно) | Радіус |
|-----|---------|-------------------------|--------|
| game-1 | Фортеця | 48.6734, 26.5612 | 80 м |
| game-2 | Старий міст | 48.6717, 26.5578 | 60 м |

Уточніть координати в Google Maps і змініть `center` / `radiusM`.

## Геолокація на телефоні (чому не з’явилось вікно)

1. Запустіть **`npm run dev:https`** (не звичайний `dev`).
2. Відкривайте **`https://192.168.x.x:5173/`**, не `http://` — інакше браузер **не покаже** запит GPS.
2. На сайті натисніть жовту кнопку **«Дозволити геолокацію»** (після натискання з’являється системне вікно).
3. У вікні оберіть **«Дозволити» / Allow**.

### iPhone (Safari)

- **Налаштування → Конфіденційність і безпека → Служби геолокації** — увімкнено.
- **Safari** — «Під час використання» або «Запитувати».
- Якщо раніше натиснули «Заборонити»: **Налаштування → Safari → Додатково → Дані веб-сайтів** — знайдіть IP сайту і скиньте, або **Налаштування → Safari → Розташування** → «Запитувати».
- У Safari на сторінці: **аА** (ліворуч у адресному рядку) → **Налаштування веб-сайту → Розташування → Дозволити**.

### Android (Chrome)

- **Налаштування → Додатки → Chrome → Дозволи → Місцезнаходження** — «Дозволити».
- На сторінці: замок біля адреси → **Дозволи → Місцезнаходження → Дозволити**.

### Без GPS на столі

У блоці **«Тест геолокації (dev)»** оберіть «Біля фортеці» / «Біля мосту» — Unity не потрібен.

## Як перевірити геолокацію

### 1. Кнопки на сайті (найпростіше)

При `npm run dev` на головній є блок **«Тест геолокації (dev)»**:

- **Біля фортеці** — відкривається лише гра 1  
- **Біля мосту** — лише гра 2  
- **Далеко (Київ)** — обидві заблоковані  
- **Справжній GPS** — реальна позиція телефона/ПК  

### 2. Chrome DevTools

F12 → **Sensors** → Location → **Other** → введіть lat/lng з таблиці вище → оновіть сторінку → **Справжній GPS** у dev-панелі.

### 3. Телефон на місці

`https://192.168.x.x:5173` у тій самій Wi‑Fi — реальний GPS біля фортеці/мосту.

Unity-білди **не переписуються** — замок лише на кнопці «Грати» у веб-хабі.

## GitHub Pages

Сайт буде за адресою: **https://iddoll.github.io/Kamianets_Deer/**

### Перший раз (один раз у GitHub)

1. Закомітьте і запуште код у `main` на https://github.com/iddoll/Kamianets_Deer
2. **Settings → Actions → General** → блок **Workflow permissions** → увімкніть **Read and write permissions** → **Save**
3. **Settings → Pages** → **Source:** **GitHub Actions** (не «Deploy from branch»)
4. Після push у `main` вкладка **Actions** → workflow **Deploy to GitHub Pages** (зелена галочка)
5. Сайт: https://iddoll.github.io/Kamianets_Deer/

**Тест геолокації на опублікованому сайті** (без поїздки в Кам’янець):

https://iddoll.github.io/Kamianets_Deer/?test=1

Там з’являться кнопки «Біля фортеці» / «Біля мосту».

### Якщо Actions падає з помилкою 403 (checkout)

Це не код сайту, а права токена на GitHub:

1. **Settings → Actions → General → Workflow permissions** → **Read and write permissions**
2. Перезапустіть workflow: **Actions** → останній запуск → **Re-run all jobs**

Червоний **«pages build and deployment»** можна ігнорувати — це старий спосіб Pages; потрібен лише **Deploy to GitHub Pages**.

### Локальна перевірка перед пушем

```bash
npm run build:ghpages
npm run preview
```

Відкрийте URL, який покаже `preview` (з префіксом `/Kamianets_Deer/`).

### Плюси GitHub Pages для вас

- **HTTPS** — на телефоні з **будь-якого Wi‑Fi / 4G** працює справжня геолокація (кнопка «Дозволити геолокацію»)
- Не потрібен `npm run dev` і `192.168…`
- WebGL-білди в `public/builds/` потрапляють у збірку автоматично

### Оновлення сайту

Кожен `git push` у гілку `main` перезбирає і публікує сайт.

> Великі Unity-білди: GitHub обмежує розмір репозиторія; якщо білди дуже важкі, краще Git LFS або окремий хостинг для `.wasm` / `.data`.
